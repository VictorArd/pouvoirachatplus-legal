Declaration d'Accessibilite

Application PouvoirAchat+ — Etablie le 21 avril 2026

Engagement : AIRIAUD Victor s'engage a rendre l'application PouvoirAchat+ accessible conformement a l'article 47 de la loi n°2005-102 du 11 fevrier 2005.

1. Etat de conformite

L'application mobile PouvoirAchat+ est partiellement conforme au Referentiel General d'Amelioration de l'Accessibilite (RGAA) version 4.1.

2. Referentiel

  • Referentiel : RGAA 4.1 (base sur WCAG 2.1 niveau AA)
  • Technologies utilisees : Flutter (Dart), composants Material 3, Semantics API
  • Environnements de test : iOS 17+ (VoiceOver), Android 14+ (TalkBack)

4. Contenus non accessibles

4.1 Non-conformites

  • Dynamic Type : les tailles de police ne s'adaptent pas encore au reglage systeme de l'utilisateur. Les textes utilisent des tailles fixes. Correction prevue avant la version 1.1.
  • Mode haut contraste : pas de mode haut contraste dedie dans les reglages de l'app (bordures renforcees, soulignement liens). Correction prevue avant la version 1.1.

4.2 Exemptions

  • Carte interactive (flutter_map) : la carte utilise des tuiles raster et n'est pas entierement navigable au clavier/lecteur d'ecran. Une vue liste equivalente est disponible comme alternative accessible.

5. Mesures d'accessibilite mises en oeuvre

  • Audit WCAG 2.1 AA complet realise sur toutes les pages (Session 22-23)
  • Contrastes de couleurs verifies (ratio minimum 5.9:1 pour le texte courant)
  • 25 tooltips ajoutes sur les boutons d'action (IconButtons)
  • 21 GestureDetectors wrapes avec Semantics (labels descriptifs en francais)
  • Touch targets de 48dp minimum sur tous les elements interactifs
  • Support daltonisme : couleur + forme + icone + texte (jamais couleur seule)
  • Mode sombre complet
  • Vue liste alternative a la carte pour les stations
  • Boutons +/- comme alternative au pinch-to-zoom sur la carte
  • Actions principales dans la zone inferieure de l'ecran (usage a une main)
  • Langage clair (niveau B1 francais), pas de jargon technique

6. Technologies d'assistance compatibles

  • iOS : VoiceOver (iOS 17+)
  • Android : TalkBack (Android 14+)
  • Grossissement : zoom systeme iOS et Android
  • Switch Access : compatible navigation par contacteur Android

7. Etablissement de cette declaration

  • Date : 21 avril 2026
  • Methode : auto-evaluation basee sur le RGAA 4.1
  • Outils : Dart Analyzer, audit manuel VoiceOver/TalkBack, verification contrastes (WebAIM Contrast Checker)
